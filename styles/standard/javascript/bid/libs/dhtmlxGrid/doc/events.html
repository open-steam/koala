Starting from dhtmlGrid 1.3 new event system introduced
( while old methods still accessible - it is strongly recommended to use new one while creating new code )

Old event system New event system
Different name for event commands Single command for all events
Event handlers not detachable Detachable handlers



Attaching event

Instead of
grid.setSomeEventHandler(func)
You can use
var eventId=grid.attachEvent(EventName,func)
Where
  • EventName - name of event, please look below
  • func - function name or function object

The value returned by attachEvent function can be used to detach event in future
var eventId=grid.attachEvent("onClick",func);
... some actions...
grid.detachEvent(eventId);

    • onDrag
    Event occured after item was dragged and droped on another item, but before item moving processed.

    Event raised after drag-and-drop processed.

  • target grid object
  • index of column from which drag started

    Event raised if drag moved over potencial landing

    event called on start of column moving operation

    event called after column moved
  • index of moved column

    • onOpenStart
    Event raised immideatly after item in tree got command to open/close , and before item was opened//closed. Event also raised for unclosable nodes and nodes without open/close functionality - in that case result of function will be ignored.

    • Returns:true - confirm opening/closing; false - deny opening/closing;

    Event raised immideatly after item in tree got command to open/close , and before item was opened//closed. Event also raised for unclosable nodes and nodes without open/close functionality - in that case result of function will be ignored.
  • ID of node which will be opened/closed
