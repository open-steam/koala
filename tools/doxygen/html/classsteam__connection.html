function login:

The login procedure

The user logs in through an established connection. Credentials are checked by sTeam. If successful, an object of type steam_user is given back; otherwise a FALSE is returned.

Parameters:
string$pLoginUser's login name
string$pPasswordUser's password
boolean$reloginTRUE if only relogin (reduces login overhead)
Returns:
steam_user | FALSE

$login_arguments contains the following: 0 => user name 1 => server version 2 => last reboot 3 => last login 4 => pike version 5 => database 6 => root-room 7 => group-all 8 => map ( name->object ) containing modules 9 => map ( int id->factory object ) 10 => map ( config name->config value )

function predefined_command:

Defines COAL-commands in a convenient way

This function translates the arguments into a COAL-command by means of steam_request. Either it returns an instance of steam_request, or - depending on the buffer argument - the transaction id in case of a buffered command.

ATTENTION: Function is needed when you want to implement new COAL-commands!

Examples:

$my_workroom = $steam_connection->predefined_command( $steam_connection->get_current_steam_user(), "query_attribute", array( "USER_WORKROOM" ), 0 );

$transaction_id = $steam_connection->predefined_command( $my_workroom, "get_inventory", array(), 1 );

Parameters:
steam_object$pObject
string$pMethodCOAL-command, see sTeam-documentation for further informations
mixed$pArgsArray of arguments, see sTeam-documentation for further informations
boolean$pBuffer0 = send now, 1 = buffer command and send later
Returns:
steam_request | integer Depending on the buffer argument either a steam_request instance or a unique transaction id is given back
