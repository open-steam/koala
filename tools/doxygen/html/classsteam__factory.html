function create_user:

Creates a new user and returns its activation code

Please keep in mind, that you will need extended rights to execute this function, that means a steam_connector with an administrator login.

Suggestion: Divide the registration and activation process, if you want to be sure about the existence of the user's e-mail-address; send the activation-code via e-mail...

Example for registration and activation: $activation_code = steam_factory::create_user( $steam_con, "nbates", "mother", "norman@bates-motel.com", "Norman", "Bates", "english" ); if ( $activation_code ) { $new_user = steam_factory::username_to_object( "nbates" ); $new_user->set_attributes( array( "country" => "United States", "occupation" => "motel keeper" ) ); if ( $new_user->activate( $activation_code ) ) { print( "Bates, you can login now!" ); } } else { print( "Login name exists. Choose another one." ); }

See also:
steam_user->activate()
Parameters:
steam_connector$pSteamConnectorconnection to sTeam-server, for creating new users you will need extended rights
string$pLoginuser's login name
string$pPassworduser's password
string$pEMailuser's email
string$pFullnameuser's surname
string$pFirstnameuser's firstname
string$pLanguageuser's prefered language (optional)
Returns:
string activation code; needed to activate this login

function get_object:

returns an instance of steam_object or one of its subclasses

Please read the file's description of steam_factory.class.php; due to the purpose that steam_factory should be the central class for managing all instances of steam_objects, you should envoke this function for any new steam_object.

Parameters:
steam_connectionthe connection to steam is essential
intpID if you have an object-id, place it here (optional)
intpType see steam_types.conf.php for the type definitions (optional)
Returns:
steam_object new instance of a subclass of steam_object, depending on the type definition parameter
