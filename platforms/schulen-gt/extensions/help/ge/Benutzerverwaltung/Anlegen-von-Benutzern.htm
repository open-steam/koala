Anlegen von Benutzern

Als Schuladministrator können Sie Benutzer anlegen. Geben Sie dazu einfach den Vornamen gefolgt von dem Nachnamen ein, der Benutzername, unter dem der Benutzer sich anmelden kann, wird dann automatisch erzeugt, indem an den ersten Buchstaben des Vornamens der letzte Teil des Nachnamens angehängt wird. Dabei werden ggf. Sonderzeichen entfernt und u. U. eine Nummer angehängt, um den Benutzernamen eindeutig zu machen. Der Benutzername ist nachträglich nicht änderbar. Auch können Benutzer nur von der zentralen Administration, nicht aber von den Schuladministratoren gelöscht werden. Bitte beachten Sie daher die folgenden Hinweise zur Namensvergabe und legen Sie keine Testbenutzer an.

Das Kennwort des Benutzers entspricht dem Benutzernamen. Bitten Sie daher die von Ihnen angelegten Benutzer, ihr Kennwort so bald wie möglich zu ändern.

Auf dem Server wird automatisch für jeden Benutzer ein Ordner angelegt, in dem er volle Schreib- und Leserechte besitzt. Dieser Ordner dient als persönlicher Arbeitsbereich.

Tragen Sie in das erste Feld den Namen des Benutzers einfach in der Reihenfolge Vorname Nachname ein, also beispielsweise Severus Snape. Daraus wird dann der Benutzername ssnape gebildet. Falls es bereits einen Benutzer dieses Namens gibt, wird eine Nummer angehängt, also ssnape1 oder ggf. mit einer höheren Nummer. Dieses Vorgehen ist nur bei Namen empfehlenswert, die keine Umlaute oder Sonderzeichen enthalten; diese werden derzeit einfach aus dem Namen entfernt. Aus dem Namen Björn Grün wird daher der Benutzername bgrn gebildet. Um das zu vermeiden, sollten Sie den Namen in Umschrift eingeben, also als Bjoern Gruen, so dass der Benutzername bgruen gebildet wird.

Problematisch sind außerdem mehrteilige Nachnamen. Bei der Generierung des Benutzernamens wird davon ausgegangen, dass das letzte Wort des eingegebenen Namens der Nachname ist. Bei Namen wie Albus Percival Wulfric Brian Dumbledore führt dies zum gewünschten Resultat adumbledore, für Wilhelmina Grubbly Plank würde jedoch der Name wplank gebildet. In diesem Fall ist es daher ratsam, den Namen als Wilhelmina GrubblyPlank einzugeben, so dass der Benutzername wgrubblyplank gebildet wird.

Als Rolle ist Schüler vorab ausgewählt; wenn Ihr Benutzer eine andere Rolle erhalten soll, wählen Sie diese hier bitte aus. Sie sollten nicht zu viele Schuladministratoren festlegen, da dies zu einem erhöhten Koordinationsaufwand oder unübersichtlichen Strukturen führen kann.

Im Feld Hauptgruppe werden alle als solche wählbaren Gruppen angezeigt. Welche das sind, können Sie in Absprache mit unseren Ansprechpartnern festlegen. Wir haben für Ihre Schule vorab zwei Hauptgruppen eingerichtet, eine für die Lehrer sowie eine für die Schule selber. Letztere ist für alle Benutzer Ihrer Schule gedacht, die keine Lehrer sind. Zusätzlich können Sie den neu anzulegenden Benutzer in eine oder mehrere weitere Gruppen eintragen, die an Ihrer Schule von Ihnen oder einem Ihrer Kollegen eingerichtet wurden.

Zwischen den Hauptgruppen und den weiteren Gruppen besteht kein wesentlicher, sondern nur ein technischer Unterschied. Es muss lediglich sichergestellt sein, dass jeder Benutzer auf jeden Fall einer Gruppe zugeordnet ist; diese eine Gruppe muss eine Hauptgruppe sein. Darüber hinaus kann ein Benutzer in beliebig vielen Hauptgruppen und weiteren Gruppen Mitglied sein. Nähere Informationen zur Mitgliedschaft in Gruppen und wie Sie diese Zuordnung vornehmen können, finden Sie im Abschnitt »Mitglieder zuordnen«.

Nachdem Sie alle erforderlichen Angaben gemacht und auf den Button »Benutzer anlegen« geklickt haben, wird angezeigt, welchen Benutzernamen der neue Benutzer erhält. Dieser Name ist für den Server eindeutig. Sie können nun auf den Benutzernamen klicken, um sich die zu dem Benutzer gespeicherten Daten anzeigen zu lassen und ggf. erforderliche Änderungen vorzunehmen. Sollten Sie sich bei der Angabe der Rolle oder der Hauptgruppe vertan haben, können Sie so sofort Korrekturen vornehmen. Falls Sie den Namen des Benutzers entsprechend der vorstehenden Empfehlungen in modifizierter Form angegeben haben, um die Generierung des Benutzernamens zu beeinflussen, sollten Sie hier auch den Namen des Benutzers auf seinen tatsächlichen Namen ändern. Auch eine E-Mail-Adresse können Sie hier angeben. Näheres zu den Daten des Benutzers finden Sie in den Abschnitten »Ihre Daten« und »Benutzer suchen«.

Beachten Sie bitte die Benutzervereinbarung für den Server, wenn Sie neue Benutzer anlegen, und weisen Sie die neuen Benutzer darauf hin, dass sie mit der Beantragung einer Benutzerkennung diese Benutzerordnung akzeptieren. Bedenken Sie insbesondere, dass es nicht zulässig ist, Benutzerkennungen zu erstellen, die nicht erkennbar an eine reale Person gebunden sind. Benutzerkennungen für bestimmte Funktionen, wie beispielsweise eine möglicherweise von mehreren Benutzern verwendete Kennung für einen Schuladministrator sind nur nach ausdrücklicher, dokumentierter Genehmigung durch unsere Ansprechpartner zulässig. Eine Ausnahme stellen Zugangskennungen für Schulklassen, Kurse und Arbeitsgemeinschaften dar, die von einem Lehrer verantwortlich betreut werden. In diesem Fall hat der Schuladministrator dafür Sorge zu tragen, dass der verantwortliche Lehrer jederzeit ermittelt werden kann

Derzeit können über die webbasierte Oberfläche nur einzelne Benutzer angelegt werden; wir hoffen, bald auch das Anlegen von Benutzerkennungen in Listenform anbieten zu können. Falls Sie viele Benutzer anlegen möchten, senden Sie uns bitte eine Textdatei, die den Namen der Hauptgruppe trägt, in die Benutzer eingetragen werden sollen. Die Datei sollte dann zeilenweise die Namen der Benutzer enthalten in der gleichen Form enthalten, wie sie sie in das Textfeld des Formulars zur Einrichtung eines einzelnen Benutzers eingeben würden. Bitte achten Sie darauf, dass Sie uns nur die Namen von Benutzern zusenden, die noch keine Benutzerkennung für den Server haben.
